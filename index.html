Apresentação da Semana

Bem-vindo à semana que transformará suas páginas estáticas em experiências dinâmicas. Até agora, você se tornou um arquiteto de estruturas com HTML e um designer de interfaces com CSS. Agora, vamos instalar o "sistema nervoso" de nossas páginas com o JavaScript (JS).

Introdução ao JavaScript: O Cérebro da Operação

O que é o JavaScript?

O JavaScript (JS) é a linguagem de programação que permite implementar itens complexos em páginas web, controlando o comportamento e a interatividade.

A Tríade do Front-End:
  • HTML: A estrutura (esqueleto).
  • CSS: A apresentação (roupas).
  • JavaScript: O comportamento (cérebro).

Como Inserir JavaScript

A abordagem profissional é usar um arquivo externo, linkado no final do <body> para garantir que o HTML seja carregado primeiro.

<!-- No final do seu arquivo .html -->
<script src="js/script.js"></script>

Fundamentos da Linguagem

Variáveis com let e const

Use const por padrão para valores que não mudam e let para valores que podem ser reatribuídos. Evite var.

const PI = 3.14159; // Não pode ser alterado
let contador = 0;      // Pode ser alterado
contador++;

Template Literals (`)

Para construir strings com variáveis, use crases (` `). É a forma moderna, limpa e legível.

const usuario = "Cleziel";
// O jeito moderno:
console.log(`Olá, ${usuario}! Bem-vindo ao curso.`);

O DOM: O Mapa da Página

O Document Object Model (DOM) é como o JavaScript "enxerga" o HTML. Ele transforma a página em uma árvore de objetos que podem ser manipulados.

Selecionando e Alterando Elementos

Usamos métodos como document.getElementById() ou document.querySelector() para "capturar" um elemento e depois modificar suas propriedades.

// Captura o elemento com id="titulo"
const meuTitulo = document.querySelector('#titulo');

// Altera seu texto e cor
meuTitulo.innerText = "Título Alterado por JS!";
meuTitulo.style.color = '#1a5e3a'; // Verde IFES

Eventos: Reagindo às Ações

Eventos são ações do usuário (clique, envio de formulário, etc.). Usamos addEventListener para "ouvir" esses eventos e executar uma função em resposta.

const meuBotao = document.querySelector('#meu-botao');

meuBotao.addEventListener('click', function() {
  alert('Você clicou no botão!');
});

Segurança: A validação no front-end (JS) melhora a experiência do usuário, mas a validação de segurança real e obrigatória deve sempre ser feita no back-end.

Boas Práticas e Padrões de Mercado

Dicas para um Código Profissional:
  • Organização: Separe HTML, CSS e JS em arquivos diferentes.
  • Nomenclatura: Use camelCase para variáveis e funções (ex: nomeUsuario).
  • Comentários: Comente o "porquê" da sua lógica, não o "o quê".
